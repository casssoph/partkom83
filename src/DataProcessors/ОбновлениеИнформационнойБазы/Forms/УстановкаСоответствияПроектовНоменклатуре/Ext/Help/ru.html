Обработка «Установка соответствия проектов и номенклатуры»

Обработка используется в контексте обновления на ред. 10.3

В редакции 10.3 изменены принципы учета реализации в разрезе проектов. Из табличной части документов реализации убрана колонка "Проект", вместо этого используются следующие механизмы:

  • В периодическом регистре сведений Установка проектов для номенклатуры организуется связь между номенклатурой и проектом; при этом при проведении документа реализации актуальный на момент проведения проект подставляется в движения по регистру "Продажи" автоматически.
  • На закладке "Дополнительно" документа реализации возможно явно указать проект, на который будет отнесены все движения регистра "Продажи" по этому документу; данные регистра Установка проектов для номенклатуры в данном случае не используются.

Таким образом, если в редакции 10.2 использовалось разделение продаж по проектам, перед началом работы в редакции 10.3 необходимо заполнить регистр Установка проектов для номенклатуры. Обработка позволяет использовать два варианта заполнения этого регистра:

  • только значения, актуальные на дату: в регистр будут помещена только информация о соответствии проекта номенклатуре, которое использовалось при последнем по отношению к указанной дате отражении реализации по данной номенклатурной позиции;
  • за период: в регистр будут помещена информация о соответствии проекта номенклатуре, указанному при каждой реализации по данной номенклатурной позиции за выбранный период. Поскольку количество записей здесь будет гораздо большим, чем при первом варианте заполнения, обработка может занять продолжительное время. При большом количестве документов реализации и выборе второго варианта заполнения регистра Установка проектов для номенклатуры рекомендуется разбивать выполнение обработки на несколько сеансов, каждый раз выбирая очередной период для выборки данных.
