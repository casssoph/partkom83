Инструкция по выгрузке балансов из 1с83 на сайт.

1.       Основные тезисы:

- инициация процедуры обмена выполняется на стороне сайта

- обмен выполняется с помощью web-сервиса 1с83 GetExchangeMessage

- параметр ExchangePlan при обращении к web-сервису должно быть равен "credit"

- регламент обмена (частота обращений к сервису) на данный момент: каждые 2 минуты

 -cкорость расчета в среднем составляет 3 - 5сек

- для автонумерации сообщений в 1с83 создан план обмена ‘Обмен с сайтом: данные покупателей’, где

        узел с id = 6 – это отправитель (база 1с83),

узел с id = 5 – это получатель (Сайт), у которого должен быть выставлен признак ‘исходящий’

параметр Sender при обращении к web-сервису по "credit" должен быть равен 5.

- на сайт выгружаются только изменения, т.е. только те балансы, которые изменились с момента последнего подтвержденного сайтом сообщения

- данные по балансам рассчитываются на основании

        РегистрНакопления.Взаиморасчеты и РегистрНакопления.ДепозитыКонтрагентов

- история по выгруженным на сайт балансам хранится в

        РегистрСведений.ВзаиморасчетыДляПланаОбменаССайтом.

Регистр автоматически очищается от устаревших данных таким образом, чтобы на GUID договора была одна последняя запись, подтвержденная сайтом. В регистре так же фиксируются время обращения к сервису, момент времени, на который выполнялся расчет и время получения подтверждения, что полезно в случае анализа проблем с обменом.

 

 

2.       Условия отбора Договоров:

- ВидДоговора = Перечисления.ВидыДоговоровКонтрагентов.СПокупателем

- ДоговорПриостановлен = ЛОЖЬ

- КА (Владелец договора) не является ‘нашим’

- КА не является магазином, т.е. не заполнен ГоловнойКонтрагент

- КА обладает Логином, которые запрашиваются из Справочник.УчетныеЗаписиСайта

- Владелец.ЮрФизЛицо = Перечисления.ЮрФизЛицо.ЮрЛицо

- Договор должен иметь ссылку на GUID договора 1с7, для чего опрашивается

 РегистрСведений._ДляПереносаДанных

 

3.       Структура сообщения

Сообщения обмена формируются в соответствии со схемой xml, описанной в XDTO «СхемаОбменаПартКом83_Сайт» с типом «РегистрыНакопления.Взаиморасчеты». На сайт передаются:

- UID договора 77

- сумма баланса

где

сумма баланса – это сводная сумма по всем договорам 1с83, ссылающихся на один и тот же GUID 1с7

 

URIПространстваИмен = http://ws-02.part-kom.ru/partkom83/hs/SiteExchange/XMLSchema. Путь прописан явным образом в менеджере ПланыОбмена.ОбменПартКом83_Сайт.URIПространстваИмен().

 

4.       Местонахождение программного кода

Обработка выгрузки сообщения выполняется в менеджере соответствующего плана обмена, т.е. в

ПланыОбмена.ОбменПартКом83_Сайт_данные_покупателей.ВыгрузитьСообщениеОбмена_Взаиморасчеты, куда передаются в качестве параметров id узла получателя и номер принятого на сайте сообщения.

 

5.       Оповещение

В случае обнаружения проблем (в выгрузку попали ‘магазины’, для договора 1с83 с изменившимся балансом не найдена ссылка на UID 1с7) выполняется рассылка по e-mail, в соответствии с настройками в
